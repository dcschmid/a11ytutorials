Es ist sehr wertvoll, wenn Sie von Benutzern mit Behinderungen lernen. Es ist etwas anderes, eine Liste mit Punkten zu lesen und
umzusetzen. Es ist etwas ganz anderes, wenn man eine Behinderung selbst erlebt. Wenn man keine Behinderung hat, kann man das nicht
nachvollziehen. Man kann nur raten. Mit der Zeit werden Sie besser verstehen, was Sie tun müssen. Aber es ist trotzdem gut, wenn Sie
sich Feedback von Menschen mit Behinderungen holen.

## Beheben Sie zuerst, was Sie wissen, wie man es behebt

Überprüfen Sie Ihre Website, bevor Sie Rückmeldungen von Menschen mit Behinderungen einholen. Sie wollen nicht, dass die Person
lange nach offensichtlichen Problemen mit der Barrierefreiheit suchen muss. Achten Sie besonders auf:

- Laufen Sie die Seiten mit dem Tool axe durch und beheben Sie alle Fehler.

- Sie können mit der Tabulatortaste durch die Seite navigieren und Funktionen aktivieren. Testen Sie, ob die Seite auch ohne Maus
  funktioniert. Nur wichtige Funktionen wie Schaltflächen, Links und Formularelemente sollten mit der Tabulatortaste ausgewählt werden
  können.

Sie können noch viel mehr tun. Sie brauchen keine Person mit Behinderung, die Ihnen sagt, dass Ihnen altem Text auf einem Bild oder
einem Formularelement fehlt. Automatisierte Tools können das. Bitten Sie die Person, Ihnen mehr Feedback zu geben.

## Achten Sie auf Usability-Feedback

Benutzer mit Behinderungen haben wahrscheinlich Probleme mit der Barrierefreiheit und der Benutzerfreundlichkeit. Viele werden
sagen: "Das ist verwirrend." Oder: "Ich weiß nicht, wie es geht." Vielleicht sehen Sie eine Person, die einen Screenreader benutzt.
Sie bewegt sich überall herum, findet aber nicht, was sie sucht. Es ist zwar technisch zugänglich, aber nicht so organisiert, wie
erwartet.

Angenommen, alle Rückmeldungen haben nur einen Grund: Die Benutzerfreundlichkeit ist schlecht. Und es gibt keine Fehler in den
Richtlinien zur Barrierefreiheit. Müssen Sie die Benutzerfreundlichkeit verbessern, um zu sagen, dass Ihre Website barrierefrei ist?

Sie haben alle Regeln zur Barrierefreiheit erfüllt. Das ist ein Grund zum Feiern! Das ist toll.

Gehen Sie ins Detail. Ist das das Beste, was Ihre Website kann? Stellen Sie sich vor, Sie wären behindert. Wie schwierig war es für
diese Person? Würde das Design für Sie selbst als gut gelten, wenn Sie die gleiche Behinderung hätten? Sind Sie damit zufrieden?

<Aside title="Die Benutzerfreundlichkeit wirkt sich auf die wahrgenommene Barrierefreiheit einer Website aus">
  Wenn eine Website schwer zu bedienen ist, wissen Menschen mit Behinderungen oft nicht, ob das an der Barrierefreiheit liegt oder
  an anderen Problemen. Wenn sie nicht wissen, wie sie Ihre Website nutzen können, denken sie vielleicht, dass sie nicht zugänglich
  ist, auch wenn sie den technischen Standards entspricht, aber für Menschen mit Behinderung schwer zu bedienen ist. Machen Sie die
  Website benutzerfreundlich, damit niemand die Barrierefreiheit in Frage stellt.
</Aside>

## Nehmen Sie Bildschirmvideos von Menschen mit Behinderungen auf, die die Website nutzen

Sie können testen, wie benutzerfreundlich Ihre Website ist, indem Sie eine Person mit Behinderung darauf testen lassen.

1. Wählen Sie einen Bereich Ihrer Website aus, den Sie auswerten möchten.

   - Produkte suchen, auswählen und in den Warenkorb legen.

   - Artikel im Warenkorb kaufen.

   - Benutzerprofil erstellen.

   - Sitzplatz im Flugzeug auswählen.

   - Computer mit passender Hardwarekonfiguration kaufen.

   - Lernsimulation in einem Online-Kurs nutzen.

   - Geld von einem Konto auf ein anderes übertragen.

2. Laden Sie eine Bildschirmaufzeichnungssoftware auf den Computer der Person.

3. Richten Sie eine separate Videokamera ein, um die Handlungen und Gesichtsausdrücke der Person aufzuzeichnen. So erhalten Sie
   Daten, die in einem Bildschirmvideo nicht sichtbar sind.

4. Bitte nehmen Sie die Person auf, die während der Navigation von ihren Gedanken und Reaktionen erzählt. Fragen Sie die Person, was
   sie macht, wenn sie etwas macht. Notieren Sie alles, was sie macht, auch die Tastenanschläge. Fragen Sie, was sie erwartet und was
   sie nicht erwartet. Notieren Sie alles. Fragen Sie, ob sie so etwas oft macht und ob sie gerne experimentiert. Es ist besser, wenn
   Sie nicht im Raum sind, während die Person das Video aufnimmt. So können Sie ihr nicht helfen, wenn sie nicht weiterkommt.

5. Schauen Sie sich die Videos an und notieren Sie, was schwierig war.

6. Sprechen Sie mit dem Benutzer über seine Erfahrungen. So erhalten Sie weitere Informationen.

## Welche Arten von Behinderungen sollten Sie einbeziehen?

In einer perfekten Welt ohne Geld- und Zeitdruck würden wir alle möglichen Meinungen einholen. Aber das ist nicht realistisch.
Sie sollten trotzdem Rückmeldungen von Menschen mit verschiedenen Behinderungen einholen. Hier erfahren Sie, was Sie von Menschen
mit verschiedenen Behinderungen lernen können.

- Benutzer von Screenreadern können Ihnen helfen, Ihre Texte und Videos besser zu machen.

- Benutzer von Bildschirmlupen können Ihnen helfen, Ihre Sehschwäche zu verstehen. Sie können Ihnen zum Beispiel sagen, wie Sie
  Ihre Website gestalten sollten, damit sie besser lesbar ist.

## Jeder Mensch mit einer Behinderung ist einzigartig

Schließen Sie nicht von Ihren eigenen Erfahrungen oder Meinungen auf alle Menschen mit Behinderung. Nehmen Sie das Feedback ernst,
aber denken Sie daran, dass es nicht die einzige Wahrheit ist. Man kann auch eine andere Person fragen. Das sollten Sie machen. Es
gibt viele verschiedene Dinge zu beachten.

- Die Technologiekompetenz und -erfahrung der Person

- Die Vorerfahrung der Person mit Ihrer Website

- Der Grad der Erfahrung der Person im Umgang mit ihrer assistive Technologie (z. B. Screenreader, Bildschirmlupe)

- Die Zeit, die die Person die Behinderung hat

- Ob die Person eine Behinderung oder mehrere Behinderungen hat

- Das Wohlbefinden der Person und die bisherige Erfahrung bei der Bereitstellung von Benutzerfeedback

- Die Fähigkeit der Person, sich gut zu erklären

- Das Maß an Kreativität der Person, wenn sie mit einer Zugangsbarriere auf einer Webseite konfrontiert wird

- Die Beziehung der Person zu dir

All das beeinflusst, wie du Feedback bekommst. Manchmal ist eine gute Feedback-Sitzung alles, was du brauchst. Manchmal brauchst
du mehrere Sitzungen, um alles zu verstehen.
