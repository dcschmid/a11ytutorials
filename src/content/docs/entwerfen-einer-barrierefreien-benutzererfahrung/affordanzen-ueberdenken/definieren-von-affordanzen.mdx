## James J. Gibsons ursprüngliches Konzept der Affordanzen

Der Psychologe James J. Gibson prägte 1977 den Begriff "Affordanzen". Damit bezeichnet er alle Handlungen, die jemand mit einem
bestimmten Objekt ausführen kann. Hier sind ein paar Beispiele:

<Aside type="tip" title='Beispiel: Die Affordanzen von Türen'>
  Eine Tür ist eine Öffnung in einer Wand. Man kann durch sie von einer Seite zur anderen gehen. Eine Tür kann auf verschiedene
  Arten geöffnet werden.
</Aside>

<Aside type="tip" title='Beispiel: Die Affordanzen von Kaffeebechern'>
  Eine Kaffeetasse kann man zum Halten von Dingen verwenden, vor allem von Flüssigkeiten. Wenn sie einen Henkel hat, kann man sie
  an einem Ort greifen, wo sie nicht heiß ist. Man kann sie auch an einem Haken aufhängen.
</Aside>

## Affordanzen für wen?

Gibson sagt, dass die Möglichkeiten, die ein Gegenstand bietet, davon abhängen, was man mit ihm machen kann. Das ist bei jedem
Menschen anders. Was bedeuten Affordanzen? Es gibt nicht eine einzige Antwort. Man muss sich fragen: Für wen sind Affordanzen da?

Hier kommt das Konzept der Affordanzen ins Spiel. Es geht darum, wie etwas zugänglich ist.

<Aside type="tip" title='Beispiel: Die ungleichen Affordanzen von Treppen'>
  Wie gut man eine Treppe nutzen kann, hängt davon ab, wie beweglich man ist.

  - Eine Person, die sich frei bewegen kann: Treppen helfen dabei, von einem Stockwerk zum anderen zu gehen, entweder nach oben oder
    nach unten.

  - Für Rollstuhlfahrer sind Treppen nutzlos, weil sie nicht von einer Ebene zur anderen wechseln können. Treppen können für
    Rollstuhlfahrer schwierig sein. Sie müssen zum Beispiel darauf achten, dass es mehrere Ebenen gibt. Außerdem müssen sie nach
    oben oder unten schauen, um zu wissen, wo sie hin müssen. Aber Treppen sind nicht das Wichtigste an einem Gebäude. Treppen
    sind nicht für Rollstuhlfahrer gemacht.
</Aside>

<Aside type="tip" title='Beispiel: Die ungleichen Möglichkeiten von Computerbildschirmen'>
  Wie gut man auf einem Computerbildschirm sieht, hängt davon ab, wie gut man sieht.

  - Eine sehende Person benutzt einen Computer, um damit zu arbeiten. Sie sieht einen Bildschirm mit Symbolen und kann mit der
    Maus oder dem Finger darauf klicken.

  - Für blinde Menschen sind Computerbildschirme nicht nützlich. Sie können keine Bilder auf dem Bildschirm sehen und auch nicht
    mit Bildern auf dem Bildschirm interagieren.
</Aside>

## Affordanzen sind binär (nach Gibson)

Ob Affordanzen eines Objekts existieren, hängt davon ab, ob man das Objekt und die Person berücksichtigt. Auch wenn eine Person nicht
alle Affordanzen eines Objekts kennt, hat das Objekt trotzdem alle Affordanzen. Objekte haben Affordanzen, egal ob die Person sie
erkennt oder nicht.

<Aside type="tip" title='Beispiel: Die sekundären (unbeabsichtigten) Affordanzen eines Volleyballs'>
  Volleybälle werden mit den Händen geschlagen. Das ist das Wichtigste an einem Volleyball. Aber für die meisten Menschen kann man
  damit noch viel mehr machen.

  - Ein Volleyball kann man treten.

  - Man kann auf einem Volleyball sitzen.

  - Ein Volleyball kann im Notfall als Schwimmhilfe verwendet werden.

  - Ein Volleyball kann ein Fenster einschlagen. Das ist aber nicht gut.

  - Man kann ein Gesicht auf einen Volleyball malen und ihn als Kopf einer Vogelscheuche verwenden. Das ist aber nicht üblich.

  Sie wissen, worum es geht. Ein Volleyball kann viel mehr als nur Volleyball. Gibsons Konzept der Affordanzen erklärt all diese
  Möglichkeiten.
</Aside>

## Donald Normans Konzept der Mensch-Computer-Interaktion (HCI) von Affordanzen

Ab 1988 begann Donald Norman, den Begriff "Affordanzen" in der Gestaltung von Computer-Benutzeroberflächen zu verwenden. Er
verwendete den Begriff ähnlich wie Gibson, aber mit einigen wichtigen Unterschieden. Laut Norman gilt:

- Objekte können nur dann wirken, wenn man sie auch wahrnehmen kann. Wenn jemand eine Affordanz nicht wahrnehmen kann, existiert
  sie für diese Person nicht. Gibson sagt, dass die Affordanz existiert, egal ob man sie wahrnehmen kann oder nicht.

- Wie etwas wahrgenommen wird, hängt von der Kultur, dem Hintergrund, der Bildung und anderen Erfahrungen ab.

- Objekte können so gestaltet werden, dass ihre Funktionen klar erkennbar sind. Oder aber das Design kann Funktionen verdecken
  oder einschränken. Für Norman ist die Qualität des Designs wichtig, damit ein Objekt die gewünschten Eigenschaften hat.

Donald Norman hat die Idee der Affordanzen weiterentwickelt. Er wollte, dass Objekte so gestaltet sind, dass sie die richtige
Art von Affordanzen bieten.

<Aside type="tip" title='Gutes Beispiel: Ein Türgriff mit offensichtlichen "drücken" - und "ziehen" Affordanzen'>
  Türen sind ein gutes Beispiel für Designaffordanz.

  Wenn eine Tür nur in eine Richtung aufgeht, muss man eine Seite drücken und die andere ziehen. Die Tür sollte so gestaltet sein,
  dass man gleich erkennt, welche Seite man drücken und welche ziehen muss. Wenn Sie auf einer Seite eine "drückbare" Metallplatte
  und auf der anderen Seite einen "ziehbaren" Griff anbringen, müssen die Menschen nicht nachdenken, welche Aktion sie ausführen
  sollen. Die Menschen führen die Aktion aus, die ihnen die Seite der Tür bietet.
</Aside>

<Aside type="danger" title='Schlechtes Beispiel: Ein Türgriff mit mehrdeutigen oder irreführenden Möglichkeiten'>
  Wenn Sie eine Tür mit "ziehbaren" Griffen auf beiden Seiten entwerfen, denken die Leute, dass sie die Tür öffnen, indem sie den
  Griff ziehen, selbst wenn sie ihn drücken müssen. Norman sagt, dass die Tür so designed ist, dass die Leute denken, sie können
  sie auf beiden Seiten ziehen. In Wirklichkeit ist aber nur eine Seite ziehbar.
</Aside>

## Webdesign-Affordanzen

Man kann Webdesigns auch danach untersuchen, was sie bieten und was nicht.

<Aside type="danger" title='Schlechtes Beispiel: Bearbeitbare Texteingaben, die nicht bearbeitbar aussehen'>
  Sie müssen Ihren Namen, Ihre E-Mail-Adresse und Ihre Telefonnummer eingeben. Die Felder haben keine Rahmen oder andere
  Markierungen. So kann man nicht erkennen, wo man klicken muss, um den Text einzugeben.

  <Picture src={Form} formats={['avif', 'webp']} alt="" />

  Die Texteingaben befinden sich nicht wie erwartet direkt unter den Beschriftungen, sondern weiter oben. Das kann für manche
  Benutzer überraschend sein. Auch wenn der Benutzer richtig rät, warum sollte er dann überhaupt raten? Die Texteingaben sollten
  so gestaltet sein, dass man sie gut erkennen kann.
</Aside>

Wenn das Design nicht gut durchdacht ist, können Menschen mit Behinderung es nicht so nutzen, wie andere Menschen. Das gilt vor
allem für benutzerdefinierte JavaScript-Steuerelemente.

<Aside type="danger" title='Schlechtes Beispiel: Ein benutzerdefinierter Link ohne Link-Möglichkeiten für einige Benutzer'>
  Der folgende Link sieht und verhält sich wie ein normaler Link. Das heißt, dass man ihn anklicken kann und er einen auch dorthin
  bringt, wo er hinführt. Für Mausbenutzer mit Sehfähigkeit ist das alles wahr.

</Aside>

## Physische vs. kognitive Affordanzen

Professor Rex Hartson, ein Informatikexperte, unterscheidet zwischen physischen und kognitiven Affordanzen. Eine Notausgangstür ist
ein Beispiel für eine physische Affordanz. Das Schild mit der Aufschrift "Ausgang" ist ein Beispiel für eine kognitive Affordanz. Es
zeigt uns, dass wir das Gebäude verlassen können. Ein Beispiel für eine kognitive Affordanz im Webdesign ist das blaue Auszeichnen
von Links. Benutzer erwarten, dass Text, der so formatiert ist, ein Link ist. Die physische Struktur kennzeichnet den Link als solchen.
Die Benutzer können den Link mit der Maus oder der Tastatur aktivieren.

## Entwerfen unter Berücksichtigung von Affordances

Für unsere Zwecke in diesem Modul hilft uns Donald Normans Erklärung weiter. Er sagt, dass wir die Bedürfnisse der Nutzer in unseren
Webdesigns berücksichtigen sollten. Wir wollen, dass alle Nutzer unsere Designs verstehen, auch Menschen mit Behinderung.

Wahrnehmbare Angebote: Die Benutzer müssen sehen können, was sie mit einem Element machen können. Jeder Mensch nimmt Dinge anders
wahr.

- Eine sehende Person erkennt Affordanzen vor allem, wenn sie sie sieht.

- Eine blinde Person erkennt Affordanzen über den Ton (über einen Screenreader) und die Namen und Rollenwerte der Elemente im Webdesign.

- Eine taubblinde Person nimmt Affordanzen hauptsächlich durch Berührung wahr. Wie blinde Menschen entdecken sie Affordanzen durch
  die Struktur des Webdesigns.

- Eine gehörlose Person erkennt Affordanzen vor allem, wenn sie etwas sieht. Meistens kann sie die Benutzeroberfläche gut nutzen. Aber
  sie kann keine Videos und Audios anschauen, wenn es keine Untertitel oder Transkripte gibt.
