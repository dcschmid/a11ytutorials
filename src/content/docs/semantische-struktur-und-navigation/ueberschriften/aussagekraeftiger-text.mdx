## Überschriften müssen klar und aussagekräftig sein

Überschriften sollten aussagekräftig sein, damit Benutzer wissen, worum es geht. Wenn Benutzer von Bildschirmleseprogrammen die Liste der
Überschriften aufrufen, hören sie die Ebene der Überschrift sowie den Text innerhalb der Überschrift.

<Aside type="tip" title="Gutes Beispiel: Beschreibende Überschriften">
  So kann ein Artikel über Notfallvorbereitung aufgebaut sein:

```html live
<h1>Emergency Preparedness Guide</h1>
<h2>Know the Risks</h2>
<h2>Make a Plan</h2>
<h2>Get an Emergency Kit</h2>
<h3>Emergency Kit Basic Items</h3>
<h2>Resources</h2>
```

</Aside>

<Aside type="tip" title="Schlechtes Beispiel: Allgemeine Überschriften">
  Die Überschriften erklären nicht, was in den einzelnen Abschnitten steht. Die Angabe "Abschnitt 1" ist zwar korrekt, aber
  für Benutzer von Bildschirmleseprogrammen nicht hilfreich.

```html live
<h1>Emergency Preparedness Guide</h1>
<h2>Section 1</h2>
<h2>Section 2</h2>
<h2>Section 3</h2>
<h3>(a)</h3>
<h2>Section 4</h2>
```
</Aside>

## Die Überschrift sollte kurz und einprägsam sein.

Eine Überschrift sollte nicht zu lang sein, damit die Nutzer sie gut finden können.
